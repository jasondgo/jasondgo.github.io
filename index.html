Lechuga

Automated Food Service

Lechuga is an automated food system designed to prepare and deliver bowl-based meals with minimal human involvement. The system integrates multiple subsystems including conveyors, hoppers, robotic arms, and a bowl dispenser coordinated by a central controller. Prototyping has been carried out using 3D-printed components, motors, sensors, and pneumatic actuators to achieve precise motion and ingredient dispensing. Control is managed through Teensy 4.1 microcontrollers communicating over CAN bus, with redundancy and error-handling logic built in to maximize reliability. Customer orders and payments are handled through a kiosk interface, and the platform can integrate with as many kiosks as needed for efficient order taking and completion.

Customer Ordering Interface

Web Application

I developed both the user-facing interface and the backend logic for a customer kiosk, ensuring seamless communication between the frontend, backend, and the automated systems that process orders. Collaborating with a freelancer, I helped design the UI in Figma and generated HTML and CSS for the interface using AI-assisted tools, which I then vetted, debugged, and integrated into the kiosk application. On the backend, I used Node.js wrapped in Electron.js to handle HTTP requests from the frontend and manage order processing, including communication with automated systems to complete orders and safely respond to error commands. I also integrated the Stripe API for payment processing with error handling and designed the system to scale seamlessly, supporting as many kiosks as needed simultaneously. Additionally, I implemented order history tracking and uploaded data to a database for record keeping. - Jason Gomez

Order and Automation Controller

System Logic and Coordination

I created the central controller for the automated food system, managing the location and fulfillment of each order to ensure the correct ingredients were assembled in every bowl. The controller coordinated communication between the kiosk frontend, the Teensy motor controller, and human operators, sending high-level instructions to the MCU to complete orders and handling errors as they arose. I applied principles of concurrent programming to manage multiple simultaneous tasks and integrated a CAN bus for communication with the system’s microcontroller. I developed data structures to facilitate order communication between both the kiosk frontend and the central controller, as well as between the automated system MCU and the central controller. Additionally, I implemented an algorithm to determine bowl movement distances and select which hoppers to dispense ingredients from during each cycle, and I designed robust error-handling procedures that used APIs to notify operators when issues occurred. - Jason Gomez

Embedded Motor Controller and Actuator

Automated System Control

I used Teensy microcontrollers to control stepper motors, actuators, and receive input from sensors to operate a robotic arm, bowl dispenser, food dispensers, and conveyor system in a coordinated manner. I implemented stepper motor control using hardware peripheral timers and applied acceleration and deceleration curves to ensure smooth motion. CAN bus communication was established with the Order and Automation Controller to synchronize actions across subsystems, which were coordinated using state machine logic. I designed and integrated sensors to monitor food levels and used linear Hall-effect sensors to implement zeroing mechanisms for the conveyor and robotic arm. Additionally, I developed a CMake build system to support cross-compiling and flashing the project onto the Teensy 4.1. - Jason Gomez

Actuator Controlller and IO Expansion PCB

Automated System Electrical

I developed a custom PCB using a Teensy 4.1 microcontroller and MCP23017 I/O expanders to control a system of over 15 stepper motors. The system required precise multi-speed operation and synchronized motion across all motors, which was achieved through optimized I²C communication and expansion. To support reliable power delivery, the board incorporated a 24V-to-5V voltage regulator capable of handling both supply voltages, improving overall system stability. Filtering capacitors were added to suppress noise and voltage ripple, ensuring clean power for sensitive digital and analog components. To enhance robustness, the PCB included redundant power-failure protection, short-circuit safeguards, and overcurrent detection. All microcontrollers and ICs were mounted on headers rather than soldered directly, allowing for rapid replacement and reducing downtime during repairs. The modular design also supported scalability, allowing additional MCP23017 expanders to be integrated for future system growth. Careful attention was given to grounding, trace routing, and power distribution to maintain signal integrity under heavy loads. - Alexis Mejia

Zeroing Mechanism PCB

Linear Hall Effect Sensor PCB

I designed a compact PCB integrating a DRV5055 Hall effect sensor for magnetic field detection and signal output. The sensor operates on a regulated 3.3V supply and provides an analog output voltage proportional to the magnetic field strength, enabling precise measurement and real-time monitoring of nearby magnetic sources. A polarized capacitor was added for power supply filtering, reducing noise and stabilizing the sensor’s performance under fluctuating load conditions. The circuit was designed with a screw terminal input for straightforward connection to external systems and mounting holes for easy mechanical integration. Attention was given to grounding and power routing to minimize interference and ensure accurate sensor readings. - Alexis Mejia
